Cube Sticker Math — Bilinear Interpolation

This document explains how to divide any perspective quad (a cube face) into an N×N grid of perfectly flat sticker polygons, with no bowing, no bulging, and no gaps between faces.

1. The Problem with Simple Approaches

A cube face in a perspective or 3D-rendered image is a general quadrilateral — four corners that do not form a perfect parallelogram. This causes two naive approaches to fail:

ApproachWhat goes wrong
dx/dy step vectorsVectors computed per face drift apart at shared edges — faces don't align
Midpoint of diagonal midpointsOnly correct for parallelograms. On a trapezoid the "center" sinks or bulges off the face plane

The correct solution is bilinear interpolation — the standard technique used in texture mapping, image warping, and 3D graphics to place points inside a quad.

3. The Bilinear Interpolation Formula

For parameters u (0→1, left to right) and v (0→1, top to bottom), any point inside the quad is:

P(u, v) = (1-v) × [ (1-u)×TL + u×TR ]
+ v × [ (1-u)×BL + u×BR ]

Applied separately to x and y:

x = (1-v) × ((1-u)×TL.x + u×TR.x) + v × ((1-u)×BL.x + u×BR.x)
y = (1-v) × ((1-u)×TL.y + u×TR.y) + v × ((1-u)×BL.y + u×BR.y)

You can think of it as: interpolate along the top edge, interpolate along the bottom edge, then interpolate between those two results.

4. Generating an N×N Grid

For a grid of N×N stickers, each sticker at position [row, col] (0-indexed) has its four corners at:

Corneruv
Top-left col / N row / N
Top-right (col+1) / N row / N
Bottom-right (col+1) / N (row+1) / N
Bottom-left col / N (row+1) / N
Changing N is the only thing needed to go from a 2×2 to a 4×4 to a 5×5 cube. The four corner coordinates never change.

5. Why Shared Edges Always Align

When two faces share an edge, they share the same two corner points on that edge. Because every sticker corner is computed purely from those shared corners via the same formula, the edge points are identical floating-point values — not approximations. Gaps are geometrically impossible.

Example — the Front face top edge and the Top face bottom edge both pass through front(u, 0) and top(u, 1). Since both faces share the same two corner points at that edge, every interpolated point is identical.

6. The Python Generator

Drop in your 8 cube corners and change N to any size:

def bilerp(TL, TR, BL, BR, u, v):
    x = (1-v)*((1-u)*TL[0] + u*TR[0]) + v*((1-u)*BL[0] + u*BR[0])
    y = (1-v)*((1-u)*TL[1] + u*TR[1]) + v*((1-u)*BL[1] + u*BR[1])
    return (round(x, 1), round(y, 1))

# ── Your 8 cube corners ──────────────────────────────────────────
# Label:  A=top-back-left   B=top-back-right
#         C=top-front-left  D=top-front-right
#         E=bot-front-left  G=bot-front-right  H=bot-back-right
A=(246,41);  B=(499,73)
C=(96,109);  D=(384,154)
E=(109,397); G=(379,458);  H=(493,342)

# ── Face functions (TL, TR, BL, BR per face) ────────────────────
def front(u,v): return bilerp(C, D, E, G, u, v)
def top(u,v):   return bilerp(A, B, C, D, u, v)
def right(u,v): return bilerp(D, B, G, H, u, v)

# ── Grid size: change this one number ───────────────────────────
N = 4   # 2=2x2  3=3x3  4=4x4  5=5x5  etc.

# ── Generate SVG polygons ────────────────────────────────────────
def pt(p):        return f"{p[0]},{p[1]}"
def poly(a,b,c,d): return f"{pt(a)} {pt(b)} {pt(c)} {pt(d)}"

for face_name, face_fn in [("FRONT", front), ("TOP", top), ("RIGHT", right)]:
    print(f"<!-- {face_name} -->")
    for row in range(N):
        for col in range(N):
            tl = face_fn( col/N,     row/N    )
            tr = face_fn((col+1)/N,  row/N    )
            br = face_fn((col+1)/N, (row+1)/N )
            bl = face_fn( col/N,    (row+1)/N )
            print(f'<polygon points="{poly(tl,tr,br,bl)}"/>')
    print()

8. Face Corner Assignments

FaceTLTRBLBR
Front (F)CDEG
Top (U) ABCD
Right (R)DBGH
Direction matters. The u-axis on the Right face runs front→back (D→B), which is the reverse of the Top face's v-axis (also runs front→back, C→A reversed). This is why shared edges verify correctly only when you account for traversal direction.

9. Extending to Hidden Faces

The same formula applies to the three hidden faces. Their corners are either known cube corners or can be inferred by continuing the geometry:

FaceTLTRBLBR
Left (L) ACinfer*E
Bottom (D)EGinfer*infer*
Back (B) ABinfer*infer*

*Inferred corner = known corner + down-vector, where down = E - C (the vertical step of the front face).
